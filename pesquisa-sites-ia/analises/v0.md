## 4. Color Palette

### Geist Design System Colors (Vercel's Official Design System)

v0 uses Vercel's Geist design system. Based on the design system documentation:

| Color | Hex | Usage |
|-------|-----|-------|
| **Background (Dark)** | `#000000` | Primary dark mode background |
| **Background (Light)** | `#FFFFFF` | Primary light mode background |
| **Foreground (Dark)** | `#EDEDED` | Text on dark backgrounds |
| **Foreground (Light)** | `#171717` | Text on light backgrounds |
| **Gray 100** | `#F7F7F7` | Subtle backgrounds |
| **Gray 200** | `#E5E5E5` | Borders, dividers |
| **Gray 400** | `#A3A3A3` | Placeholder text |
| **Gray 600** | `#666666` | Secondary text (dark mode) |
| **Gray 900** | `#171717` | Primary text (light mode) |
| **Blue (Accent)** | `#0070F3` | Vercel brand blue, links, CTAs |
| **Error Red** | `#EE0000` | Error states |
| **Success Green** | `#00A550` | Success states |
| **Warning** | `#F5A623` | Warning states |

### Color System Notes
- Uses **oklch color format** for shadcn/ui components
- CSS variables defined in `tokens.css` for theming
- Both light and dark themes supported
- "Black and white precision" is the core aesthetic principle

## 5. Typography

### Geist Font Family

| Element | Font | Details |
|---------|------|---------|
| **Sans-Serif (Primary)** | Geist Sans | Created by Vercel with Basement Studio, designed for legibility and simplicity |
| **Monospace** | Geist Mono | Used for code blocks and technical content |
| **CSS Variables** | `--font-sans`, `--font-mono`, `--font-serif` | Preconfigured with Tailwind/shadcn |

### Geist Font Characteristics
- Designed to embody "simplicity, minimalism, and speed"
- "Precision, clarity, and functionality at its core"
- Available as open-source (Google Fonts, npm package `geist`)
- Sans-serif designed for maximum legibility at all sizes
- Clean, geometric letterforms

### Font Loading
```
npm install geist
```
Via next/font for Next.js projects (optimized loading)

## 6. Visual Effects

### Interface Design
- **Minimal, functional interface** - The product interface itself is the visual
- **VS Code-style editor** - Split-pane layout with code editor and live preview
- **Real-time preview** - Live application rendering as AI generates code

### Theme System
- **Light and Dark mode** - Full theme switching support
- **Design Mode Preview** - Users can adjust colors for design systems in both light and dark mode

### Animation
- Smooth transitions between states
- Code generation animation (text appearing in editor)
- Preview updates with smooth rendering

### Visual Philosophy
- **Ultra-minimal** - Vercel's design DNA: black, white, and precision
- **Content-first** - The generated UI IS the visual showcase
- **No decorative gradients** - Unlike Linear, v0 keeps visual effects purely functional
- **Clean borders and spacing** - Generous whitespace, clear component boundaries

## 9. Tone of Voice

- **Technical but approachable** - Speaks to developers primarily
- **Understated** - Vercel's brand voice is confident without being loud
- **Precise** - "design, iterate, and scale" - each word chosen carefully
- **Collaborative** - "Your collaborative AI assistant" - positions v0 as a partner
- **Professional** - No slang, no exclamation marks
- **Vercel DNA** - The brand voice inherits Vercel's engineering-focused credibility
- **Developer-first** - Mentions React, Next.js, Tailwind CSS, shadcn/ui explicitly

---

## 10. How They Talk About AI (Without Cliches)

v0's approach to describing AI capabilities:

- **"Collaborative AI assistant"** - Not "revolutionary AI" or "intelligent AI"
- **"Design, iterate, and scale"** - Three concrete verbs, no abstract promises
- **"Build Agents, Apps, and Websites"** - AI agents are listed alongside concrete outputs
- **"Agentic workflows"** - Technical term, used naturally without hype
- **"Describe what you want in plain English"** - Focuses on the user's natural language, not AI's intelligence
- **"Production-ready React code"** - Emphasizes the concrete output quality
- **"Full-stack applications for the web"** - Grounded in real engineering outcomes

**Key pattern**: v0 never claims AI is "smart" or "intelligent." Instead, it describes concrete outputs (React code, Next.js apps, Tailwind CSS components) and positions AI as an assistant following the user's direction. The emphasis is always on what gets produced, not how magical the AI is.

---

## 11. Key Design Takeaways for FlowAI

1. **Extreme minimalism** - Black, white, and precision typography
2. **Geist font family** - Purpose-built typography reinforces brand identity
3. **Product IS the hero** - The prompt interface and generated UI are the visual showcase
4. **No decorative elements** - Zero unnecessary visual flourish
5. **Developer credibility** - Explicit mention of technical stack (React, Next.js, Tailwind)
6. **Collaborative language** - "Your assistant", not "our AI"
7. **Vercel ecosystem** - Brand strength comes from the Vercel brand association
8. **Open-source foundation** - Design system (Geist), component library (shadcn/ui) all open
9. **Rebranding signals maturity** - v0.dev to v0.app indicates platform evolution
10. **February 2026 updates** added Git integration, VS Code editor, database connectivity - evolving from component generator to full development platform
