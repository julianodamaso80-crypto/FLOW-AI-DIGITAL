## 3. Color Palette

Cursor uses a distinctive warm dark palette that avoids the cold blues typical of developer tools:

| Role | Color Name | Hex |
|------|-----------|-----|
| Primary Light | Cararra (Off-White) | `#F7F7F4` |
| Primary Dark | Rangitoto (Deep Charcoal) | `#14120B` |
| Secondary Dark | Dark Olive Brown | `#26251E` |
| Background Dark | Near-Black (warm) | Deep warm black tones |
| Text Light | Off-White / Cream | Warm white tones |
| Accent | Likely warm neutrals | Consistent with brand warmth |

**Design philosophy:** "Minimal, dark, and developer-centric, pairing warm off-white tones with deep charcoal backgrounds." The palette deliberately uses warm darks (brown-blacks) rather than cool darks (blue-blacks), giving the site a unique warmth among developer tools.

**Key distinction:** Unlike VS Code's cool dark theme or GitHub's neutral grays, Cursor's palette has a subtle warmth — the off-white (#F7F7F4) has a yellow/cream undertone, and the dark (#14120B) has a warm brown undertone rather than pure black.

## 4. Typography

| Usage | Font Family | Notes |
|-------|-------------|-------|
| Brand / Headlines | **Cursor Gothic** | Custom typeface by Kimera (Munich) |
| Code / Monospace | **Cursor Mono** (in development) | Custom monospace companion |
| System / UI | **Berkeley Mono** | Used in code editor interface |
| Fallback | System sans-serif | Standard fallbacks |

### Cursor Gothic Details
- **Created by:** Kimera (Munich-based type foundry)
- **Characteristics:** Slightly more condensed than standard grotesques, squarish shapes with higher contrast, creating smoother rhythm in large amounts of text and at small sizes
- **Evolution:** The partnership with Kimera "initially focused on custom type but eventually expanded to the entire brand redesign"
- **Philosophy:** The typeface is described as "the typographic core" of Cursor's identity — the brand is built around the typeface, not the other way around
- **Application:** Used across the homepage, marketing materials, blog, and documentation — consistency from "app icon to code editor"

## 5. Visual Effects

### Scenic Painted Landscape Backgrounds
The most distinctive visual element: Cursor uses **painted landscape wallpapers** as artistic backdrops for their interface demonstrations. These are scenic, artistic paintings — not photographs or abstract tech art.

### Interactive Product Demonstrations
- **Live/animated IDE demonstrations** showing Cursor in action
- **Slack integration demos** — showing team collaboration
- **GitHub integration demos** — showing code review and debugging
- **Smooth transitions** between demo states

### Other Effects
- **Subtle dark-to-light gradients** in section backgrounds
- **Clean card-based layouts** for feature sections
- **Minimal animation** — the demos ARE the animation; no gratuitous particle effects
- **No grain, no blur, no WebGL** — the artistic wallpapers provide all the visual interest needed
- **Focus on product screenshots** as the primary visual element

## 6. Image/Video Style

- **Painted landscape wallpapers** — The signature visual element. Art-quality landscape paintings serve as backdrops for IDE screenshots, creating a striking contrast between artistic beauty and technical precision
- **Product interface screenshots** — Real Cursor IDE shown in context
- **Integration screenshots** — Slack, GitHub, terminal interfaces
- **No human photography** on the homepage
- **No abstract AI art or generative visuals**
- **The artistic wallpaper choice** communicates: "coding can be beautiful" and "we care about aesthetics" — differentiating from the typically utilitarian developer tool aesthetic

## 9. How They Talk About AI Without Cliches

Cursor takes a uniquely practical approach to AI language:

**What they avoid:**
- "Revolutionary" / "game-changing" / "disruptive"
- Sci-fi or futuristic language
- Explaining how AI works under the hood
- Abstract promises about AI's potential
- Robot or humanoid AI imagery

**Instead they use:**
- "The best way to code with AI" — Simple, comparative, practical
- "Built to make you extraordinarily productive" — Benefit is human productivity, not AI capability
- "Predicts your next action" — Functional description, not hype
- "Learns how your codebase works" — Practical, relatable capability
- "You control how much independence to give the AI" — The autonomy slider concept; user is always in control
- "Accelerate development" — Action verb + clear outcome
- "Tab completion, Cmd+K, full autonomy" — Technical specificity replaces vague AI claims

**Key insight:** The "autonomy slider" metaphor is brilliant — it reframes AI not as a binary (AI does it / human does it) but as a spectrum of control. This removes the fear/hype and makes AI feel like a tool setting, like adjusting volume.

## Key Takeaways for FlowAI Digital

1. **Painted landscape wallpapers as backgrounds** — This is the most creatively daring choice among all five sites. Cursor proves that a developer tool can use fine art as a design element. FlowAI could consider using unexpected, high-quality imagery (perhaps Brazilian landscapes or architectural photography) as section backgrounds instead of generic stock photos.

2. **Custom typography as brand foundation** — Cursor Gothic is the brand. When you build identity around type, everything else falls into place. FlowAI should invest in choosing (not necessarily creating) a distinctive typeface that becomes synonymous with the brand.

3. **Warm dark palette** — The warm-toned darks (#14120B, #26251E) are significantly more inviting than pure black (#000000). FlowAI's dark theme should use warm darks, not cold ones.

4. **"Autonomy slider" as a framework** — This mental model for AI control is directly applicable to FlowAI's sales engineering pitch. "You control how much we do" — from diagnostic to full implementation to ongoing operation.

5. **Product demos as visual content** — Instead of telling clients what you do, show the actual system working. Cursor's homepage IS a product demo. FlowAI should consider showing real CRM dashboards, real metrics, real processes.

6. **Bold, confident positioning** — "The best way to code with AI" makes a clear claim without qualifiers. FlowAI could adopt similar confidence: "The best way to engineer sales processes."

7. **Fortune 500 social proof** — Cursor prominently displays enterprise logos. FlowAI should develop equivalent social proof for its niche (dental clinic logos, testimonial numbers, result metrics).
