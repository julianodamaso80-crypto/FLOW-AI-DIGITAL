## ARSENAL DE FERRAMENTAS — QUANDO E PARA QUÊ USAR CADA UMA

### FASE: DESCOBERTA (O que as pessoas estão buscando?)

| Ferramenta | Para quê | Quando usar |
|------------|----------|-------------|
| **Google Trends** | Ver se um tema está subindo, caindo ou estável. Identificar sazonalidade. | SEMPRE antes de decidir sobre qual tema escrever. |
| **AnswerThePublic** | Mapear TODAS as perguntas reais que as pessoas fazem sobre um tema. | Antes de definir a estrutura de qualquer artigo ou página. |
| **Google Autocomplete** | Ver o que o Google sugere quando alguém começa a digitar. | Para descobrir long tails que ferramentas pagas não capturam. |
| **People Also Ask (PAA)** | Coletar as perguntas relacionadas que aparecem na SERP. | Para cada keyword alvo — viram seções do conteúdo. |

### FASE: VALIDAÇÃO (Vale a pena investir nesse tema?)

| Ferramenta | Para quê | Quando usar |
|------------|----------|-------------|
| **Semrush** | Volume de busca, dificuldade (KD), CPC, SERP features. | Para TODA keyword antes de decidir trabalhar ela. Sem Semrush, sem decisão. |
| **Semrush — Keyword Gap** | Descobrir keywords que concorrentes ranqueiam e você NÃO. | Mensalmente para encontrar frutas baixas. |
| **Análise manual da SERP** | Pesquisar a keyword no Google e analisar os 10 primeiros resultados. | Para TODA keyword antes de criar conteúdo. |

### FASE: TÉCNICA (O site está saudável?)

| Ferramenta | Para quê | Quando usar |
|------------|----------|-------------|
| **Google PageSpeed Insights** | Diagnóstico de Core Web Vitals: LCP, FID/INP, CLS. | Antes do lançamento + mensalmente + após qualquer deploy. |
| **Google Search Console** | Monitorar indexação, cobertura, erros, consultas, CTR, posição média. | Diariamente para detectar problemas cedo. |
| **Screaming Frog / Sitebulb** | Crawl completo do site: URLs, status codes, redirects, canonicals, meta tags. | Antes do lançamento + trimestralmente para auditoria completa. |
| **Bing Webmaster Tools** | Indexar no Bing. Se não existe no Bing, ChatGPT não encontra. | Imediatamente após lançar o site. |

## FRAMEWORK COMPLETO: O MÉTODO DANIH

### FASE 1: DIAGNÓSTICO
```
1. Auditoria técnica completa do site
2. Core Web Vitals (PageSpeed Insights)
3. Análise de E-E-A-T
4. Verificar indexação (Search Console)
5. Verificar Bing (Bing Webmaster)
6. Entidade Digital: site + GBP + redes formam ecossistema?
7. Competitive gap analysis (Semrush)
8. Striking distance scan: páginas posição 4-10 para otimizar?
```

### FASE 2: FUNDAÇÃO TÉCNICA
```
1. Arquitetura de URLs limpa e hierárquica
2. Schema markup em TODAS as páginas
3. Sitemap.xml correto e submetido (Google + Bing)
4. Robots.txt revisado
5. Internal linking strategy definida (pillar → clusters)
6. On-page SEO checklist aplicado em cada página existente
```

### FASE 3: PESQUISA E CONTEÚDO
```
PARA CADA CONTEÚDO NOVO:

1. DESCOBERTA
   → Google Trends: tema está subindo ou caindo?
   → AnswerThePublic: quais perguntas as pessoas fazem?
   → Google Autocomplete + PAA: quais variações existem?

2. VALIDAÇÃO
   → Semrush/Ubersuggest: volume, KD, CPC
   → Keyword Gap: concorrentes ranqueiam e eu não?
   → SERP Analysis manual: quem tá no top 10? Que formato?
   → DECISÃO: volume > 100/mês E KD compatível? SIM → avança.

3. PLANEJAMENTO
   → Classificar intenção: "como/qual/onde/por que/quanto"
   → Definir formato baseado na SERP
   → Desenhar mapa semântico: H1 → H2s → H3s
   → Definir schema markup da página
   → Definir CTAs, internal links e FAQs

4. CRIAÇÃO
   → Conteúdo 10x melhor que o top 1 atual
   → E-E-A-T: autor real, bio, credenciais, fotos, dados originais
   → On-page checklist completo
   → Link-worthy by design: dados originais, infográficos, templates

5. PUBLICAÇÃO
   → Timing: publicar ANTES do assunto explodir
   → Solicitar indexação imediata no Search Console
   → Indexar no Bing
   → Internal links de páginas existentes apontando pro novo conteúdo
```
